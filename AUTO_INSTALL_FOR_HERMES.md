# 【节点引擎】AI图片制作自动安装说明

请严格执行本文件，为当前 Hermes/节点安装“AI图片制作”能力。不要只总结内容，不要让普通用户选择技术路线。

收到这个仓库链接，就表示用户希望你直接安装默认图片能力。

## 执行原则

1. 默认该安装的就直接安装；
2. 不要先问用户要装哪个模型；
3. 不要把底层工具选择抛给普通用户；
4. 安装后必须自动验证；
5. 验证失败就说失败原因，不能假装成功。

## 立即安装

如果已经在仓库目录中，直接执行：

```bash
python3 installer/install_commercial_image_router.py
```

如果还没有仓库，先克隆再安装：

```bash
git clone https://github.com/ZRH-Iris/ai-image-creation-node-capability.git
cd ai-image-creation-node-capability
python3 installer/install_commercial_image_router.py
```

如果用户发送的是 raw 安装方式，也可以执行：

```bash
python3 <(curl -fsSL https://raw.githubusercontent.com/ZRH-Iris/ai-image-creation-node-capability/main/installer/install_commercial_image_router.py)
```

## 默认安装内容

默认安装“基础生图能力”，包括：

- 图片制作 Skill；
- 本地生图运行环境；
- 默认主生图模型；
- 基础生图工作流；
- 启动脚本；
- 安装后 smoke test。

普通用户不需要理解或选择这些内容。安装器会自动处理。

如果当前 Hermes/节点已经有可复用的 ComfyUI 或模型文件，安装器应优先复用已有内容，不重复下载；只有找不到可用模型或校验不通过时才重新下载。`--dry-run` 也要明确报告“将复用”还是“将下载”，方便正式安装前确认。

## 什么时候再装增强能力

默认安装完成后，先让用户开始做图。

如果之后遇到以下需求，再按需安装增强包：

- 短中文文字直接生成在图里；
- 古诗图、中文文化图；
- 明确要求模型本身写中文；
- 商品图精细抠图；
- 高清放大/修复；
- 更复杂的图片编辑。

这些增强包不要在默认安装时全部塞进去。

## 安装成功后回复用户

安装成功后，请这样回复，不要输出大段技术日志：

```text
【节点引擎】AI图片制作能力已经安装成功，可以开始使用了。

你现在可以直接发给我：
1. 一句想生成图片的需求；
2. 一张要处理的图片；
3. 标题、文案、尺寸或风格要求。

我会根据需求自动选择合适的图片路线，并在交付前检查结果。
```

## 安装失败后回复用户

如果失败，请简单说明是哪一类问题，例如：

- 当前机器算力不足；
- 磁盘空间不足；
- 网络无法访问 GitHub 或模型源；
- 模型下载不完整；
- 依赖安装失败；
- smoke test 没有生成图片。

最后告诉用户：

```text
你可以把这段失败提示转发给我，我会继续帮你处理。
```

## 安全边界

不得用于制作冒充他人、侵权、欺骗身份、违法营销或未经授权使用他人肖像/品牌的图片。
